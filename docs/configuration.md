# Configuration Guide

This document describes how to configure the AntiBridge Telegram Remote.

## Environment Variables (`.env`)

Create a `.env` file in the root directory of the project by copying `.env.example` and filling in the values.

| Variable | Description | Required | Default |
|----------|-------------|----------|---------|
| `TELEGRAM_BOT_TOKEN` | The bot token generated by [@BotFather](https://t.me/BotFather) on Telegram. | **Yes** | None |
| `TELEGRAM_CHAT_ID` | Your Telegram Chat ID. Get this from [@userinfobot](https://t.me/userinfobot) or similar. For security, only requests from this ID are processed. | **Yes** | None |
| `WS_PORT` | The local port used by the Express/WebSocket server to capture stream logs. | No | `8000` |
| `CDP_PORT` | The Chromium debugging port used to communicate with the IDE. | No | `9000` |
| `ANTIGRAVITY_PATH` | Full absolute path to your `Antigravity.exe` file. If left blank, the bot will attempt to automatically find it. | No | *Auto-detected* |
| `AVAILABLE_MODELS` | Comma-separated list of models to display in the `/model` switch list on Telegram. | No | Standard set of models |
| `DISABLE_SAFE_ROLLBACK` | Set to `true` to protect your local file edits from being automatically reverted/rolled back. | No | `false` |

## Windows Helper Scripts

The project includes several batch and VBScript files in the root folder to automate the startup and shutdown processes on Windows:

### 1. Initial Setup
- **`SETUP.bat`**: Installs NPM dependencies for both root and backend directories. Always run this script first.

### 2. Startup Scripts
- **`START_ALL_SILENT.vbs`**: Launcing the bot in the background silently.
  - Automatically kills active, locked Antigravity processes.
  - Launches Antigravity with the `--remote-debugging-port=9000` option enabled.
  - Runs the bot backend server in the background (no cmd window shown).
- **`START_BOT.bat`**: Launches the bot server in the foreground, displaying the output console. Useful for debugging and viewing logs.

### 3. Shutdown Scripts
- **`KILL_SERVER.bat`**: Scans for and kills active bot servers, NodeJS background runner scripts, and frees port `8000`.
- **`KILL_SERVER_ADMIN.vbs`**: Triggers `KILL_SERVER.bat` with elevated Administrator privileges to ensure locked ports are freed.

## Troubleshooting Port Conflicts
If you receive the error `Port 8000 in use` or `Port 9000 in use`:
1. Run `KILL_SERVER_ADMIN.vbs` to kill background processes.
2. Verify in Task Manager that no stray `node.exe` or `Antigravity.exe` processes are running.
3. You can change `WS_PORT` in `.env` to another port (e.g., `8080`) if port `8000` is reserved by another system service.
